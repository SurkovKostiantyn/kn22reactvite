# Перехід з npm на Yarn

Цей документ описує кроки, які були виконані для переходу проекту з пакетного менеджера **npm** на **Yarn**.

## Чому Yarn?

Yarn пропонує швидшу та надійнішу інсталяцію залежностей, краще кешування та більш детерміновану структуру `node_modules`.

## Процес міграції

1.  **Встановлення Yarn**
    Якщо Yarn ще не встановлено, його можна встановити глобально через npm:
    ```bash
    npm install -g yarn
    ```

2.  **Видалення артефактів npm**
    Щоб уникнути конфліктів, необхідно видалити файл блокування npm та папку з модулями:
    - Видалено `package-lock.json`
    - Видалено папку `node_modules`

3.  **Інсталяція залежностей через Yarn**
    Запущено команду `yarn install`, яка створила новий файл `yarn.lock`:
    ```bash
    yarn install
    ```

4.  **Оновлення конфігурації ESLint**
    Файл `eslint.config.js` було оновлено для сумісності з новим форматом Flat Config (ESLint 9+), оскільки старий формат викликав помилки.

5.  **Оновлення GitHub Actions**
    Файл workflow `.github/workflows/deploy.yml` було оновлено:
    - Замінено `npm ci` на `yarn install`
    - Замінено `npm run build` на `yarn build`
    - Змінено кешування на `cache: 'yarn'`

6.  **Оновлення .gitignore**
    Додано `package-lock.json` у `.gitignore`, щоб запобігти його випадковому додаванню в репозиторій.

## Використання

Тепер для роботи з проектом використовуйте команди Yarn:

- **Встановлення залежностей**: `yarn install`
- **Запуск dev-сервера**: `yarn dev`
- **Збірка проекту**: `yarn build`
- **Лінтинг**: `yarn lint`
