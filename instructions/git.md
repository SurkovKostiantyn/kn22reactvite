# Інструкція по роботі з Git

Ця інструкція описує основні команди Git, які знадобляться вам для роботи над лабораторними роботами.

## Основний робочий процес (Workflow)

Коли ви зробили зміни в коді і хочете зберегти їх:

1.  **Перевірка статусу**
    Подивитися, які файли були змінені.

    ```bash
    git status
    ```

2.  **Додавання змін**
    Додати всі змінені файли до підготовки (staging area).

    ```bash
    git add .
    ```

3.  **Створення коміту**
    Зафіксувати зміни з описом того, що було зроблено.

    ```bash
    git commit -m "Опис ваших змін"
    ```

    _Приклад: "Add Post component" або "Fix bug in App.jsx"_

4.  **Відправка на сервер (GitHub)**
    Завантажити ваші коміти у віддалений репозиторій.
    ```bash
    git push origin master
    ```
    _(Якщо ваша гілка називається main, використовуйте `git push origin main`)_

## Корисні команди

- **Перегляд історії комітів**

  ```bash
  git log
  ```

  Для скороченого вигляду:

  ```bash
  git log --oneline
  ```

- **Створення нової гілки**

  ```bash
  git checkout -b назва-гілки
  ```

- **Перемикання між гілками**

  ```bash
  git checkout назва-гілки
  ```

- **Злиття гілок (Merge)**
  Щоб влити зміни з іншої гілки (наприклад, `feature`) в поточну (наприклад, `master`):
  ```bash
  git merge feature
  ```

## Вирішення проблем

- **Якщо git push не працює через конфлікти**
  Це означає, що на сервері є зміни, яких немає у вас. Спочатку завантажте їх:
  ```bash
  git pull origin master
  ```
  Потім вирішіть конфлікти в коді (якщо є) і знову виконайте add, commit, push.
