## 3. Фрагмент коду з App.jsx (List Rendering)

```javascript
{mockPosts.map((post) => (
  <Post
    // Крок 4: Забезпечення унікальності ключів (key)
    key={post.id}
    id={post.id}
    title={post.title}
    content={post.content}
    author={post.author}
  />
))}
```

## 4. Що станеться, якщо видалити атрибут key?

Якщо видалити атрибут `key`, React виведе попередження в консолі розробника:
`Warning: Each child in a list should have a unique "key" prop.`

Без ключів React не може ефективно відстежувати, які елементи списку змінилися, додалися або були видалені. Це може призвести до:
1.  **Проблем з продуктивністю**: React буде перерендерити весь список замість оновлення конкретного елемента.
2.  **Помилок стану**: Якщо компоненти мають внутрішній стан (як наш лічильник лайків), цей стан може "перестрибнути" на інший елемент або бути втраченим при зміні порядку елементів.

## 5. Контрольні запитання

**1. Навіщо потрібен проп key у списках React?**
`key` допомагає React ідентифікувати, які елементи були змінені, додані або видалені. Це критично важливо для алгоритму Reconciliation (узгодження), щоб React міг ефективно оновлювати DOM, не перестворюючи всі елементи списку заново.

**2. Чому не рекомендується використовувати індекс масиву (index) як ключ?**
Використання індексу як ключа є поганою практикою, якщо порядок елементів може змінюватися (сортування, фільтрація, вставка зверху). Якщо порядок зміниться, React думатиме, що елемент з індексом `0` — це той самий компонент, що й раніше, хоча дані в ньому вже інші. Це призводить до помилок, коли старий стан (наприклад, введені дані в форму або стан checkbox) залишається на місці, хоча контент змінився.

**3. У чому різниця між імперативним створенням списку (цикл for) та декларативним (map)?**
-   **Імперативний підхід (for)**: Ми описуємо *як* отримати результат (створити порожній масив, пройтися циклом, додати елементи в масив).
-   **Декларативний підхід (map)**: Ми описуємо *що* ми хочемо отримати (новий масив, де кожен елемент є трансформацією елемента з вихідного масиву). У React ми просто кажемо: "відобрази цей масив даних як масив компонентів", і React сам займається рендерингом.

**4. Як передати всі властивості об'єкта в компонент одним рядком?**
Можна використати spread-оператор (`...`):
```javascript
<Post key={post.id} {...post} />
```
Це автоматично передасть всі поля об'єкта `post` (id, title, content, author) як окремі пропси в компонент `Post`.
